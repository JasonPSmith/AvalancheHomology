avalanche.Laplacian(M, out_deg)

Compute the out-degree Laplacian matrix of a graph.

The Laplacian is defined as

\[L = D - A\]

where D is the diagonal matrix of out-degrees and A is the adjacency matrix.

Parameters:

  • M (numpy.ndarray) – Adjacency matrix of the graph.

  • out_deg (numpy.ndarray) – Array of vertex out-degrees.

Returns:

Laplacian matrix of the graph.

Return type:

numpy.ndarray

avalanche.avalanche_complex(M, config, sinks=[], firings=1000, print_firings=False, persistence=False, return_simplices=False)

Construct the nerve complex of the avalanche complex.

Parameters:

  • M (numpy.ndarray) – Adjacency matrix of the graph.

  • config (list[int] or numpy.ndarray) – Initial sandpile configuration.

  • sinks (list[int]) – Vertices connected to the sink.

  • firings (int) – Maximum number of firing steps to simulate.

  • print_firings (bool) – If True, print firing sets during simulation.

  • persistence (bool) – If True, assign filtration values and compute persistent homology.

  • return_simplices (bool) – If True, also return a dictionary mapping simplex to firing sets.

Returns:

Either:

  • a gudhi.SimplexTree object, or

  • (SimplexTree, simplex_dictionary) if return_simplices=True.

Return type:

gudhi.SimplexTree or tuple

avalanche.avalanche_complex_original(M, config, sinks=[], steps=100, firings=1000, print_firings=False, persistence=False)

Construct the avalanche complex directly (as opposed to the nerve complex used above)

This approach is significantly slower than the nerve approach implemented in avalanche_complex above

Parameters:

  • M (numpy.ndarray) – Adjacency matrix of the graph.

  • config (list[int] or numpy.ndarray) – Initial sandpile configuration.

  • sinks (list[int]) – Vertices connected to the sink.

  • steps (int) – Number of avalanche steps to simulate before recurrence checks.

  • firings (int) – Maximum total number of firings allowed.

  • print_firings (bool) – If True, print firing sets and timestamps.

  • persistence (bool) – If True, assign filtration values for persistent homology.

Returns:

The resulting avalanche complex.

Return type:

gudhi.SimplexTree

avalanche.is_stable(out_deg, configuration)

Determine whether a sandpile configuration is stable.

A configuration is stable if every vertex contains fewer grains than its out-degree.

Parameters:

  • out_deg (numpy.ndarray) – Array of vertex out-degrees.

  • configuration (numpy.ndarray) – Current sandpile configuration.

Returns:

True if the configuration is stable, otherwise False.

Return type:

bool

avalanche.out_degrees_matrix(M, sinks)

Compute the out-degree of each vertex in a graph.

The out-degree is computed as the row sum of the adjacency matrix. Vertices listed in sinks are treated as having an additional outgoing edge to the sink vertex.

Parameters:

  • M (numpy.ndarray) – Adjacency matrix of the graph as dense NumPy array.

  • sinks (list[int]) – Indices of vertices connected to the sink.

Returns:

Array whose i-th entry is the out-degree of vertex i.

Return type:

numpy.ndarray

avalanche.run_avalanche_sandpile(L, out_deg, configuration, avalanches, display=False)

Simulate the sandpile model using parallel toppling.

The simulation proceeds by repeatedly toppling all unstable vertices simultaneously until either the requested number of avalanches has occurred or the configuration stabilises.

Parameters:

  • L (numpy.ndarray) – Laplacian matrix of the graph.

  • out_deg (numpy.ndarray) – Array of vertex out-degrees.

  • configuration (numpy.ndarray) – Initial sandpile configuration.

  • avalanches (int) – Maximum number of avalanche topplings to perform.

  • display (bool) – If True, print configurations after each toppling.

Returns:

A tuple (configs, avalanche_firing_sequence) where:

  • configs is the list of configurations encountered.

  • avalanche_firing_sequence records which vertices fired at each avalanche step as a binary vector of length |G|, where position i is 1 iff vertex i topples.

Return type:

tuple[list, list]
